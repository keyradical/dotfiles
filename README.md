# dotfiles

Cross-machine dotfiles for three hosts:

| Host     | OS         | Notes                                      |
|----------|------------|--------------------------------------------|
| `mac`    | macOS      | Work laptop. Aerospace + Hammerspoon.      |
| `pc`     | Fedora 44  | Desktop. Sway, no media keys (F1/F2/F3).   |
| `laptop` | Fedora 44  | Laptop / 2-in-1. Sway, XF86 keys, autorotate. |

The Mac is the source of truth for shared tools (nvim, tmux, zsh).

## Layout

```
config/                # shared, linked on every host
  nvim/ tmux/ zsh/ kitty/ claude/ git/
  starship.toml
  sway/ waybar/ swaylock/ wlogout/ fuzzel/ gtk-{3,4}.0/   (linked on linux hosts)
hosts/
  mac/                 # aerospace, hammerspoon, macos.sh
  pc/                  # sway PC bindings
  laptop/              # sway laptop bindings
  _linux.conf.yaml     # shared linux WM dotbot rules (sway base, waybar, ...)
install                # bootstrap: detects host, runs dotbot
install.conf.yaml      # shared dotbot rules
fedora-install.sh      # fresh-Fedora bootstrap (calls ./install)
```

Each host's `install.conf.yaml` is layered on top of the shared one by the
top-level `install` script.

## Installing

### Fresh Fedora box (pc or laptop)

```bash
git clone git@github.com:keyradical/dotfiles.git ~/.dotfiles
cd ~/.dotfiles
./fedora-install.sh
```

The script asks whether the host is a laptop / 2-in-1 / ROG ASUS / NVIDIA box,
installs dnf/dotbot/TPM, links configs, generates `~/.config/sway/config.local`
(touchpad / lid-suspend / autorotate), and prints next steps.

### Existing machine (just (re)link configs)

```bash
cd ~/.dotfiles
./install            # auto-detects: mac via uname, prompts on linux
./install pc         # or pass host explicitly
DOTFILES_HOST=laptop ./install
```

The first run writes your choice to `~/.dotfiles-host`; subsequent runs reuse it.

### Mac: migrating from the old `~/.config -> repo` symlink

If your Mac was set up with the old layout (whole `~/.config` was one symlink
to `<repo>/config`), `./install` will refuse to proceed and print exact
migration steps. The short version:

```bash
# 1. Park app-managed dirs that snuck into the repo
mkdir -p ~/.config-migration
for d in bk.yaml fish gh github-copilot htop opencode; do
    [ -e ~/.config/$d ] && mv ~/.config/$d ~/.config-migration/
done

# 2. Drop the old whole-dir symlink
rm ~/.config

# 3. Re-run
./install

# 4. Move app data back
for d in bk.yaml fish gh github-copilot htop opencode; do
    [ -e ~/.config-migration/$d ] && mv ~/.config-migration/$d ~/.config/
done
rmdir ~/.config-migration
```

## Per-host sway customisation

Shared sway config lives in `config/sway/`. Each host overrides two files,
both gitignored:

- `~/.config/sway/config.bindings` — symlink to `hosts/<host>/config/sway/bindings`. Placed automatically by dotbot.
- `~/.config/sway/config.local` — real file. Generated by `fedora-install.sh` (touchpad / autorotate / lid). Hand-edit freely; re-running the script regenerates it.

## macOS defaults

Run once per Mac: `./hosts/mac/macos.sh`.
